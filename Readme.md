# 基于 Hive 的旅游景点推荐系统

本项目是一个完整的大数据旅游推荐系统，通过爬取携程网景点与评论数据，构建 Hive 数据仓库，并基于改进的 Item‑CF 算法实现个性化景点推荐。系统采用 Django 框架提供 Web 服务，并结合 ECharts 提供多维度数据可视化。所有数据从采集、存储、离线分析到推荐生成，均在大数据平台上完成。

------

## 项目简介

针对旅游信息过载问题，本项目设计并实现了一个基于 Hive 数据仓库和协同过滤算法的旅游景点推荐系统。系统爬取携程网 1.5 万+景点和 50 万+评论，构建 Hive 四层数据仓库（ODS、DWD、DWS、ADS），完成 ETL 与离线分析；改进 Item‑CF 算法，融合评分、收藏、浏览三种用户行为，并引入热门惩罚因子，显著提升推荐准确率与多样性；基于 Django + ECharts 开发可视化 Web 系统，实现个性化推荐、城市热度排行、景点评分分布等功能；部署于 Hadoop+Spark 集群，验证了系统的可用性与推荐效果。

------

## 技术栈

- **数据采集**：Python + DrissionPage（动态爬虫）
- **数据存储**：Hive（数据仓库）、MySQL（业务数据）、HDFS（原始数据）
- **计算框架**：Hadoop + Spark（离线分析）、Hive on Spark（查询加速）
- **推荐算法**：Item‑CF（改进版）、User‑CF（对比实验）
- **后端框架**：Django（Python Web）
- **前端可视化**：ECharts、HTML/CSS/JS
- **部署环境**：Linux 虚拟机集群（三节点）

## 数据采集与处理

### 爬虫模块（`spider/`）

- **城市爬虫**（`spiderCity.py`）：抓取携程网所有省份、城市及其详情页链接。
- **景点ID爬虫**（`spiderPoiId.py`）：根据城市名称模拟搜索，动态加载并解析景点列表，获取 `poiId`、标签、等级、热度等基础信息。
- **详情爬虫**（`spiderMain.py`）：根据景点详情页 URL 爬取景点介绍、地址、电话、开放时间、图片等详细信息。
- **评论爬虫**（`spiderComment.py`）：基于 `poiId` 调用携程评论接口，爬取用户评论、评分、时间等信息。

### 数据仓库构建

- 原始数据（CSV 格式）上传至 HDFS。
- 使用 Hive 建立 **ODS**（原始数据层）、**DWD**（明细数据层）、**DWS**（汇总数据层）、**ADS**（应用数据层）四层数仓，具体代码在Hiveql文件夹下。
- 通过 Spark SQL 或 HiveQL 完成 ETL，生成各类统计表（如城市热度表、景点评分分布表、TOP10 热门景点表等），供推荐算法和前端展示使用。

## 推荐算法（改进 Item‑CF）

### 传统 Item‑CF 的不足

- 仅利用评分数据，导致矩阵稀疏，推荐结果偏向热门景点。
- 无法利用收藏、浏览等隐式反馈，用户偏好刻画不全面。

### 改进点

1. **多行为融合**
   在相似度计算中同时融入 **评分**（权重 0.5）、**收藏**（权重 0.3）、**浏览**（权重 0.2）三种行为（网格搜索确认权重），综合反映用户偏好。

2. **热门惩罚因子**
   引入景点被评论次数作为热门程度指标，在相似度公式中加入惩罚项：

   final_sim=cosinelog⁡(1+freq1)⋅log⁡(1+freq2)final_sim=log(1+freq1)⋅log(1+freq2)cosine

   降低热门景点之间的相似度，提升长尾景点的曝光机会。

### 推荐流程

1. 离线构建景点‑用户交互矩阵（评分、收藏、浏览）。
2. 计算加权余弦相似度，得到景点相似度矩阵。
3. 在线推荐时，根据用户已有行为，累加相似度 × 评分，生成候选推荐列表。
4. 若推荐数量不足，补充当天热度最高的景点，保证推荐列表长度。

## Web 系统（Django）

### 功能模块

- **用户端**
  - 登录/注册
  - 个性化景点推荐
  - 景点列表搜索、详情查看
  - 景点收藏、评论（打分）
  - 个人中心（修改信息、收藏列表、我的评论）
- **管理员端**
  - 景点信息管理（增删改查）
  - 用户管理
  - 评论审核
  - 数据统计看板

### 可视化（ECharts）

- 城市热度排行（柱状图）
- 景点评分占比（饼图）
- 景点热度分布（饼图）
- 用户评分占比（饼图）
- 景点简介词云、评论词云

### 部署与运行

1. **环境要求**

   - Python 3.8+
   - MySQL 5.7+
   - Hadoop/Hive/Spark 集群（可选，若仅运行 Web 可跳过）

2. **安装依赖**

   ```
   pip install -r requirements.txt
   ```

3. **配置数据库**
   修改 `project/settings.py` 中的 `DATABASES` 配置，指向您的 MySQL 数据库，并提前创建好数据库和表（可通过 Django 迁移或导入 SQL 脚本）。

4. **启动服务**

   ```
   python manage.py runserver 0.0.0.0:8000
   ```

5. **访问系统**

   - 用户端：`http://localhost:8000/app/login/`
   - 管理员端：`http://localhost:8000/app/adminLogin/`（默认管理员账号 admin/admin）

## 效果评估

- **数据规模**：爬取携程网 1.5 万+景点、50 万+评论，覆盖全国 300+城市。
- **推荐效果**：改进算法在准确率和召回率上均有明显提升，验证了多行为融合与热门惩罚的有效性。
- **系统性能**：在 3 节点 Hadoop+Spark 集群上，Hive on Spark 查询响应时间较原生 Hive 提升 30% 以上。
- **用户体验**：通过 Django + ECharts 构建的可视化看板，直观展示旅游数据分布，辅助用户决策。
